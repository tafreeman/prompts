01Visual Hierarchy & Core Principles

Visual hierarchy organizes elements to guide audience attention using size, color, contrast, and placement. The goal: audiences grasp the message within 3 seconds.

The 60-30-10 Rule
Color Distribution

60% primary/background, 30% supporting elements, 10% accent for emphasis. Prevents visual chaos while maintaining hierarchy.

Scanning Patterns
Z-Pattern & F-Pattern

Z-Pattern: Simple layouts. F-Pattern: Text-heavy slides - across top, down left, across again.

Signal-to-Noise
Tufte's Data-Ink Ratio

Maximize information density, minimize decorative elements. Every pixel should carry information. "Above all else, show data."

White Space Is Not Wasted Space

McKinsey, BCG, and Bain treat white space as a strategic design element. Consulting decks use generous margins and padding to signal professionalism and clarity. Tech decks use asymmetrical white space - bunching content on one side with dramatic empty space on the other for visual drama.

Key Insight

Garr Reynolds (Presentation Zen): "The highest possible signal-to-noise ratio produces the most effective slides." Every element on a slide must earn its place.

Sources: Beautiful.AI Blog, IxDF Visual Hierarchy (2026), Ramotion, Garr Reynolds, Edward Tufte

03Color Theory

Data Color Palettes

Sequential
Ordered Data

Light to dark transitions for magnitude (revenue, density, temperature). Single hue, varying lightness.

Diverging
Central Value Data

Two hues diverging from neutral center. Blue to White to Red for profit/loss, sentiment, variance from baseline.

Categorical
Distinct Groups

Visually distinct colors for unordered categories. Maximum 10 unique colors before confusion. No inherent order.

Accessibility Requirements

Color blindness affects ~8% of men and ~0.5% of women. WCAG standards require: 4.5:1 contrast ratio for standard text, 3:1 for large text/icons. Never rely solely on color to convey information - always pair with shape, pattern, or label.

ThemeBest ForConsiderations

04Layout & Grid Systems

8px Grid System

Industry standard. All spacing as multiples of 8 (8, 16, 24, 32, 40, 48px). Highly divisible, aligns with device pixel ratios. Used by Google, Microsoft, Apple design systems.

12-Column Grid

Most common for slide layouts. Divides into halves, thirds, fourths, sixths for flexible composition. Column grids for standard slides; modular grids for complex information.

Spacing Rules

📏
External >= Internal

Margins (space between elements) must be greater than or equal to padding (space inside elements) to maintain clear visual separation.

📐
Consistent Alignment

Left-align text-heavy slides. Center-align title/focal slides. Right-align rarely. Consistent alignment ties design together and establishes order.

Sources: Microsoft Layout Guide, UXDesign Spacing Article, Figma Layout Grids, Design Systems (Space, Grids, Layouts)

The Pyramid Principle (McKinsey Standard)

Developed by Barbara Minto (first female MBA hire at McKinsey). Start with the conclusion, then layer supporting evidence hierarchically. Each section supports one message. Combined with MECE (Mutually Exclusive, Collectively Exhaustive) thinking for logical argument structure.

Key Architecture Rules

One Idea Per Slide

Each slide conveys one singular assertion supported by one piece of visual evidence. Maximum 3-5 bullets if text is necessary.

Action Titles

"Q3 Revenue Down 8% YoY" not "Quarterly Results." Title states the conclusion; body provides evidence.

Progressive Disclosure

Reveal information step-by-step. Allows audience to digest before next concept. Critical for live delivery.

Sources: Assertion-Evidence.com, Harvard Catalyst, SlideGenius, McKinsey Presentation Structure (SlideModel), InkNarrates

Tufte's Core Rules

📊
Maximize Data-Ink Ratio

Proportion of ink presenting actual data vs. total ink. Remove all non-essential decoration.

🗑
Eliminate Chart Junk

No decorative graphics, 3D effects, excessive gridlines, ornamental backgrounds. Every pixel contributes to understanding.

🏷
Direct Labels > Legends

Label data directly on chart elements. Legend lookups create cognitive friction. Redesign layout to accommodate direct labels.

📝
Annotate the Insight

Data alone doesn't tell the story. Highlight the key number, add an arrow, state the insight in plain language.

Sources: Edward Tufte, Cole Nussbaumer Knaflic (Storytelling with Data), Andy Kirk, Alberto Cairo, McKinsey/BCG/Bain standards

07Reusable Slide Patterns

A taxonomy of proven slide layouts used across consulting, tech, and corporate presentations.

Layout
Card Grid (2/3/4 Column)

Rounded-rectangle panels with numbered labels, headlines, descriptions. Subtle shadows for depth. Best for: comparisons, features, services, KPIs.

Data
KPI Scorecard

4-6 key metrics with visual indicators (bars, sparklines, gauges). Color-coded RAG status. Rule: if a number doesn't drive decisions, it doesn't belong.

Flow
Process Steps

Horizontal connected shapes (hexagons, circles, arrows) showing sequential progression. Limit to 5-7 steps. Shade colors to denote movement.

Compare
Before / After

Split-screen with contrasting colors (red "before" / blue "after"). Central connector. Best for: transformations, case studies, process improvements.

Impact
Big Number Callout

Oversized statistic (60-80pt) with supporting context below. BCG standard: callout box with key metric proving the action title.

Matrix
Harvey Balls / Comparison

Grid with filled circles showing capability levels. Rows = items, columns = criteria. Standard in McKinsey, BCG, Bain feature comparisons.

Time
Timeline / Gantt

Horizontal bars showing duration, milestones, dependencies. Icons for key events. Best for: roadmaps, project plans.

Funnel
Conversion Funnel

5-level progression (Inquiry to MQL to SAL to SQL to Won). Visualizes drop-off at each stage. Best for: sales pipeline, conversion tracking.

Dashboard
4-Quadrant Layout

Slide divided into 4 equal sections (SWOT, BCG Matrix, Ansoff). Each quadrant contains different metric or analysis dimension.

Industry Conventions

Tech / SaaS

Product-led, metric-heavy. Key metrics: MRR, churn, LTV/CAC, cohort trends. Support storytelling with authentic metrics.

Consulting

Insight-led, rigorously structured. Pyramid Principle + SCQA + MECE. Black/white/blue accent. Every slide leads with conclusion. Action titles mandatory.

Government / Federal

Section 508 compliant. Proper reading order for screen readers, alt text, keyboard navigation, high contrast.

Healthcare

Evidence-based, FDA/CDC regulatory standards. Balanced risk/benefit presentation. MLR review process. Real-World Evidence (RWE) increasingly accepted.

092024-2026 Design Trends

Dark Mode + Neon

High contrast neon-on-dark palettes. Reduces eye strain, dramatic impact for keynotes and tech presentations.

Glassmorphism

Frosted-glass effect using transparency and blur. Creates depth and hierarchy. Applied to text boxes, backgrounds, layered content.

Gradient Layering

Soft aurora-style gradients replacing solid blocks. Floating panels with subtle shadows create structure without visual noise.

Asymmetric Layout

Text overlapping images, angled sections, visuals breaking grid frames. Creates editorial, premium feel while maintaining alignment.

Oversized Typography

Large fonts as primary design element. Hierarchy via size contrast, not multiple font families. Supports mobile-first/vertical slides.

Bento Grid

Inspired by Apple. Modular rectangular boxes (Japanese bento aesthetic). Replaces single-title-with-list. More organized, information-dense.

Isometric 3D

Dimensional illustrations maintaining 2D simplicity. Perfect for complex systems, process flows. Used by Slack, Dropbox.

Motion Design

Purposeful micro-animations: fades, morphing, staggered reveals. Consistent easing and timing. Based on Disney's 12 Principles of Animation.

Vertical / 9:16

Fastest-growing format in 2026. Mobile-first design. Larger fonts, less text. Responsive design thinking applied to presentations.

Sources: SlideEgg 2026 Trends, Sketchbubble 2026 Guide, Slidesgo 2026, Visible.vc 2026, 24Slides 2024, PitchWorx 2025

10Anti-Patterns - What to Avoid

Wall of Text

Slides are "glance media" (Reynolds) - audiences grasp in 3 seconds or tune out. Maximum 5 lines of text. Slides support the speaker, not replace them.

Chart Without Insight

Showing data without explaining why it matters makes the audience do the work. Always annotate with the "so what?" - highlight the key number and state the conclusion.

Bullet Point Overload

Lists of 10+ bullets overwhelm. Audience can't read and listen simultaneously. Convert to visuals. Use Assertion-Evidence model instead.

Inconsistent Formatting

Varied fonts, colors, layouts across slides signal lack of preparation. Stick to defined design system: 2-3 fonts, 5-7 colors, consistent spacing.

Stock Photo Cliches

Generic handshake photos, light bulb "innovation" images. Use small, well-taken shots with consistent style and lots of white space.

Decorative 3D Charts

3D effects distort data representation and reduce accuracy. Flat charts are always more honest. "Show data variation, not design variation" - Tufte.

Design System for Code-Gen

Color Palette

Define 5-7 named colors upfront: background, surface, text, textMuted, accent, red, green. Reference by name throughout. 60/30/10 distribution.

Font Stack

3 fonts max: title (Georgia/Montserrat), body (Calibri/Segoe UI), mono (Consolas/Fira Code). Define as constants, never inline strings.

Spacing System

Base unit of 0.15" in PptxGenJS coordinates. Standard margins: 0.8" left, 0.3" top for slide number area. Card padding: 0.15-0.2" internal.

Key Insight for Code-Generated Slides

Start with templates, not blank slides. Shape-based patterns (rectangles as cards, ovals as badges) with consistent shadow/fill create professional depth. Automate spacing, alignment, and hierarchy programmatically. The small details - consistent margins, subtle shadows, accent bar widths - separate professional from amateur output.
