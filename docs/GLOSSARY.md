# Glossary

Quick reference for domain-specific terms used throughout the `tafreeman/prompts` monorepo.

## Architecture

| Term | Definition | Where Used |
|------|-----------|------------|
| **Adapter** | Pluggable backend that implements `ExecutionEngine` protocol. Allows swapping execution strategies without changing workflow definitions. | `agentic_v2/adapters/` |
| **Adapter Registry** | Thread-safe singleton that maps adapter names (e.g., `"native"`, `"langchain"`) to `ExecutionEngine` implementations. | `agentic_v2/adapters/registry.py` |
| **Checkpoint** | Serialized snapshot of completed workflow steps, enabling resume-after-failure. Stored in SQLite via `CheckpointStore`. | `agentic_v2/adapters/native/_checkpoint_store.py` |
| **Contract** | Pydantic model defining input/output schemas for workflow steps. Contracts are **additive-only** — fields are never removed or renamed. | `agentic_v2/contracts/` |
| **Context (ExecutionContext)** | Hierarchical key-value store that flows through workflow steps. Supports scoped variables, step outputs, and expression evaluation. | `agentic_v2/core/context.py` |
| **DAG** | Directed Acyclic Graph — the execution model for workflows. Steps are nodes; `depends_on` edges define ordering. Enables maximum parallelism. | `agentic_v2/engine/dag_executor.py` |
| **Engine** | The runtime that executes workflows. Two implementations: Native (Kahn's algorithm DAG) and LangChain (LangGraph state machine). | `agentic_v2/engine/`, `agentic_v2/langchain/` |
| **Protocol** | Python `typing.Protocol` (PEP 544) defining a structural interface. All core abstractions use `@runtime_checkable` protocols for duck typing. | `agentic_v2/core/protocols.py` |
| **Step** | A single unit of work in a workflow. Defined in YAML with `name`, `agent`, `description`, `depends_on`, `inputs`, `outputs`. | Workflow YAML files |
| **Workflow** | A declarative YAML definition of a multi-step process. Steps form a DAG executed by an engine. | `agentic_v2/workflows/definitions/` |

## AI / ML Patterns

| Term | Definition | Where Used |
|------|-----------|------------|
| **BM25** | Best Match 25 — a keyword-based ranking function used alongside dense vector search for hybrid retrieval. | `agentic_v2/rag/retrieval.py` |
| **Chain of Thought (CoT)** | Prompting technique where the model reasons step-by-step before answering. Implemented as `## Reasoning Protocol` in each persona. | `agentic_v2/prompts/*.md` |
| **Chain of Verification (CoVe)** | Pattern where claims are systematically verified against sources. Used in deep research workflows. | `workflows/definitions/deep_research.yaml` |
| **Circuit Breaker** | Fault-tolerance pattern that temporarily disables a failing LLM provider after repeated errors, with adaptive cooldown. | `agentic_v2/models/smart_router.py` |
| **Confidence Gating** | Conditional execution based on a confidence score threshold. Steps with `when: ${ctx.confidence_score} < 0.8` only run if quality is insufficient. | `agentic_v2/engine/expressions.py` |
| **Embedding** | Dense vector representation of text, used for semantic similarity search. Generated by embedding models (OpenAI, Voyage, local ONNX). | `agentic_v2/rag/embedding.py` |
| **Few-Shot Examples** | Concrete input/output examples included in agent prompts to demonstrate expected behavior. | `agentic_v2/prompts/*.md` |
| **Persona** | A markdown file defining an agent's expertise, reasoning protocol, boundaries, output format, and critical rules. | `agentic_v2/prompts/` (24 files) |
| **RAG (Retrieval-Augmented Generation)** | Pipeline that retrieves relevant documents and injects them into LLM context. Full pipeline: load, chunk, embed, index, retrieve, assemble. | `agentic_v2/rag/` |
| **ReAct** | Reasoning + Acting — pattern where an agent alternates between thinking and tool use. Implemented in `BaseAgent` iteration loop. | `agentic_v2/agents/base.py` |
| **RRF (Reciprocal Rank Fusion)** | Algorithm that merges ranked lists from multiple retrieval signals (BM25 + dense) into a single ranking. | `agentic_v2/rag/retrieval.py` |
| **Rubric** | YAML-defined scoring criteria used by the evaluation framework. Dimensions include correctness, completeness, reasoning quality. | `agentic-v2-eval/` |
| **Tier** | LLM capability level used for routing. Tier 1 = fastest/cheapest, Tier 3 = most capable. Smart router selects tier based on task complexity. | `agentic_v2/models/smart_router.py` |
| **Tree of Thought (ToT)** | Pattern where multiple reasoning hypotheses are generated and evaluated in parallel before selecting the best. | `workflows/definitions/deep_research.yaml` |

## Development

| Term | Definition | Where Used |
|------|-----------|------------|
| **Additive-Only** | Schema evolution policy for contracts: new fields can be added but existing fields are never removed or renamed. Prevents downstream breakage. | `agentic_v2/contracts/` |
| **Coverage Gate** | CI threshold that fails the build if test coverage drops below a minimum (currently 70%, target 80%). | `.github/workflows/ci.yml` |
| **Expression** | String interpolation syntax (`${ctx.variable}`, `${steps.X.status}`) evaluated at runtime by the expression evaluator. | `agentic_v2/engine/expressions.py` |
| **Kahn's Algorithm** | Topological sort algorithm used by the native DAG executor. Processes steps with `asyncio.wait(FIRST_COMPLETED)` for maximum parallelism. | `agentic_v2/engine/dag_executor.py` |
| **Pre-commit** | Git hooks that run black, isort, ruff, docformatter, mypy, and pydocstyle before every commit. | `.pre-commit-config.yaml` |
| **Smart Router** | LLM request dispatcher with circuit breakers, rate-limit parsing, adaptive cooldowns, and cross-tier degradation. | `agentic_v2/models/smart_router.py` |
