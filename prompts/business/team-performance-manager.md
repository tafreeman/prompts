# Team Performance Manager

## Description

This prompt optimizes team performance through assessment, skill development planning, motivation strategies, and recognition programs. It helps managers build high-performing teams that consistently deliver results.

## Use Cases

- Team Management for Project Manager persona
- Enterprise-grade prompt optimized for production use
- Suitable for teams requiring structured, repeatable workflows

## Variables

- `[team_name]`: Team name (e.g., "Platform Engineering Team – 8 engineers")
- `[challenges]`: Current challenges (e.g., "Hero culture, slow code reviews, junior devs blocked")
- `[goals]`: Performance goals (e.g., "Reduce cycle time by 40%, improve team velocity 20%")
- `[dynamics]`: Team dynamics (e.g., "2 seniors, 4 mid-level, 2 juniors; remote-first")

## Prompt

### System Prompt

```text
You are a team performance expert who helps managers build and develop high-performing teams. You understand team dynamics, motivation theory, and practical approaches to improving delivery velocity and team health.
```

### User Prompt

```text
Develop a team performance improvement plan for [team_name].

Team Context:
- Current Challenges: [challenges]
- Performance Goals: [goals]
- Team Dynamics: [dynamics]

Please provide:
1. Team performance assessment
2. Root cause analysis of performance issues
3. Individual development priorities
4. Team process improvements
5. Motivation and recognition strategies
6. 90-day improvement roadmap
7. Success metrics and monitoring
```

## Example

### Input

```text
Develop a team performance improvement plan for Platform Engineering Team – 8 engineers.

Team Context:
- Current Challenges: Hero culture, slow code reviews, junior devs blocked
- Performance Goals: Reduce cycle time by 40%, improve team velocity 20%
- Team Dynamics: 2 seniors, 4 mid-level, 2 juniors; remote-first
```

### Expected Output

```text
## Team Performance Plan: Platform Engineering

### Current State Assessment
| Metric | Current | Target | Gap |
|--------|---------|--------|-----|
| Cycle Time | 5 days | 3 days | -40% |
| Velocity | 45 pts/sprint | 54 pts/sprint | +20% |
| Code Review Time | 48 hrs | 12 hrs | -75% |
| Junior Blocked Time | 30% | 10% | -20 pts |
| Team Health Score | 6/10 | 8/10 | +2 pts |

### Root Cause Analysis
| Symptom | Root Cause | Evidence |
|---------|------------|----------|
| Hero culture | 2 seniors do 60% of work | Commit history analysis |
| Slow code reviews | No review SLAs, async overload | Review time tracking |
| Juniors blocked | Lack of pairing, unclear escalation | 1:1 feedback |
| Low velocity | Context switching, unclear priorities | Sprint retrospectives |

### Process Improvements
| Improvement | Expected Impact | Owner | Timeline |
|-------------|-----------------|-------|----------|
| Code review SLA (24 hrs) | -50% review time | Tech Lead | Week 1 |
| Pairing sessions (2x/week) | Unblock juniors | Seniors | Week 2 |
| WIP limits (2 per person) | Reduce context switching | Scrum Master | Week 1 |
| Daily async standup | Better visibility | Team | Week 1 |

### Individual Development Priorities
| Team Member | Development Focus | Action |
|-------------|-------------------|--------|
| Senior A | Knowledge sharing | Lead 2 pairing sessions/week |
| Senior B | Delegation | Assign stories to mids, review only |
| Mid-levels | Ownership | Own features end-to-end |
| Juniors | Ramp velocity | Structured pairing, clear escalation |

### Motivation Strategies
| Strategy | Description | Frequency |
|----------|-------------|------------|
| Public wins | Shoutouts in team channel | Weekly |
| Growth conversations | Career development 1:1s | Monthly |
| Team celebrations | Sprint goal achievements | Per sprint |
| Autonomy | Let mids own architecture decisions | Ongoing |

### 90-Day Roadmap
| Month | Focus | Milestones |
|-------|-------|------------|
| Month 1 | Process foundation | SLAs live, WIP limits, pairing started |
| Month 2 | Behavior change | Review time <24 hrs, juniors unblocked |
| Month 3 | Sustain & optimize | Hit velocity target, hero culture reduced |

### Success Metrics
| Metric | Week 4 | Week 8 | Week 12 |
|--------|--------|--------|----------|
| Cycle Time | 4.5 days | 4 days | 3 days |
| Velocity | 48 pts | 52 pts | 54 pts |
| Team Health | 6.5/10 | 7.5/10 | 8/10 |
```
